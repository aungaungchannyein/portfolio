Ecommerce Api

This e-commerce application is built using Java Spring Boot in the backend, leveraging a microservices architecture for modular and scalable development. PostgreSQL serves as the primary database, while configuration servers and a discovery server (Eureka) facilitate efficient service management. The application uses an API Gateway for routing requests and secures interactions with Keycloak for authentication and authorization. Asynchronous communication is handled with Kafka, while synchronous communication is managed through OpenFeign and Rest Templates. Distributed tracing is enabled with Zipkin and Spring Actuator for better observability. The infrastructure is set up using Docker and Docker Compose for seamless deployment and management.

Key Features:

Product Catalog: Browse a wide range of products available for purchase.
User Authentication & Authorization: Secure login and registration using Keycloak for both customers and administrators.
Order Management: Seamlessly place and track orders through the system.
Real-time Product Availability: Real-time Product Availability: Check the availability of products in real-time.
Microservices Architecture: Built using Java Spring Boot with a focus on modular and scalable services.
Communication: Asynchronous communication via Kafka and synchronous interaction using Open Feign and Rest Templates.
Distributed Tracing: Monitor and trace requests with Zipkin and Spring Actuator.
API Gateway: Centralized routing and load balancing with an API Gateway.
Service Discovery: Automatically discover services through Eureka.
Infrastructure Setup: Use Docker & Docker Compose for containerization and service orchestration.
Technologies Used:
  • Java Spring Boot(Backend)
  • PostgreSQL(Database)
  • Keycloak for authentication and authorization(Security)
  • Zipkin for monitoring(Distributed Tracing)
